Casos de Uso

Caso de Uso: Registrar Usuario

El programa debe permitir el registro de cualquier usuario anonimo permitiendole, una vez registrado, el ingreso al sistema

Caso de Uso: Modificar Usuario

El programa debe permitir a cada usuario la modificacion de sus atributos, entre estos es posible eliminar el usuario por completo

Caso de Uso: Administrar Proyectos

El programa debe permitir la creacion y eliminacion de proyectos, ademas de modificar sus propiedades que son:

  1. Nombre
  2. Tipo
  3. Usuarios y permisos
  4. Configuracion de modulos

Caso de Uso: Control de Archivos

Gracias a la centralizacion que provee el IDE Web es posible integrar, de forma inteligente - incluso para algunas acciones se puede hacer de forma transparante al usuario - el sistema de control de versiones con la administracion de los archivos del proyecto, a pesar de esta integracion, la copia de trabajo de cada usuario debe estar separado del repositorio del proyecto

Caso de Uso: Control de Versiones

El programa debe tener una interfaz grafica que permita crear, acceder y actualizar repositorios de archivos a partir de las copias de trabajo de los usuarios, el manejo del repositorio se hara con una interfaz a un sistema manejador de versiones existente permitiendo aumentar la compatibilidad con otros sistemas, algunos de estos son:

  • SVN
  • CVS
  • GIT

Caso de Uso: Edicion de Codigo Fuente

El sistema debe tener una interfaz capaz de editar codigo fuente, opcionalmente incluir ayudas comunes pero muy utiles para un programador como son:

  • AutoIndentacion
  • Autocompletado de palabras
  • Opciones de Codificacion(I18N)
  • Resaltado de sintaxis

Caso de Uso: Compilacion centralizada

El sistema debe ser capaz de compilar el codigo fuente de forma centralizada (en el servidor) utilizando herramientas que determinen automaticamente que piezas deben ser recompiladas para generar los archivos ejecutables

Caso de Uso: Envio de Archivos

El servidor debe permitir al cliente diversas formas de acceder a sus archivos de forma local:

  • Descarga: Permite descargar el ejecutable y todos los archivos fuente por medio del browser

  • Sistema de archivos de Red: Utiliza un sistema de archivos de red para compartir los archivos ejecutables y de codigo fuente desde el servidor, algunos de los mas populares sistemas de archivos de red son:

    - SMB/CIFS
    - AFS
    - NFS
    - NCP
